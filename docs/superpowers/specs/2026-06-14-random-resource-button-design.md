# 随机资源「添加图片链接」按钮交互优化设计

## 背景

`liuliupi-ui/src/components/admin/webEdit.vue` 的「随机资源」标签页中，「随机头像」和「随机封面」两个子模块各有一个按钮：

- `+ 随机头像`
- `+ 随机封面`

点击后会在卡片内展开一个输入框，用户输入图片 URL 并回车/失焦后，URL 被加入对应数组。然而这两个按钮的文案和交互形式表意不清：

1. 文案中的「随机」容易被误解为「系统随机生成一张图片」。
2. 用户无法从按钮预判点击后会出现 URL 输入框。
3. 同一卡片内还有 `uploadPicture` 本地上传入口，两个入口缺乏视觉区分。

## 目标

1. 让「手动输入图片 URL」这一行为在文案和交互上都清晰可识别。
2. 让「本地上传」入口与「链接输入」入口在视觉上有明确区分。
3. 降低误操作，避免用户因回车/失焦意外添加错误 URL。
4. 保持现有后端 API 和数据模型不变。

## 约束

- 项目使用 Vue 2 + Element UI。
- 只修改 `liuliupi-ui/src/components/admin/webEdit.vue`。
- 不改动 `uploadPicture` 组件、不改动后端接口、不改动数据模型。
- 保留现有的图片网格展示和删除交互。

## 方案

### 备选方案

| 方案 | 核心思路 | 优点 | 缺点 |
|---|---|---|---|
| A | 弹窗 + 文案最小改动 | 语义直接、改动最小、明确一次「添加」动作 | 每次添加需点确认 |
| B | 卡片内分区 | 两种途径视觉完全分离 | 卡片内容变长、显拥挤 |
| C | 统一「添加图片」入口 | 入口极简 | 与本地上传原有直接体验相比多一层选择，弱化区分 |

**选定方案：A。** 它在保留现有结构的基础上，通过文案和弹窗把「添加链接」的动作说清楚，同时给上传区加标题让两种入口互不混淆，改动量最小。

## 详细设计

### 1. 按钮文案与样式

将两个按钮统一改为：

```vue
<el-button
  size="small"
  type="primary"
  plain
  icon="el-icon-link"
  @click="showAddUrlDialog('avatar')"
>
  添加图片链接
</el-button>
```

- 图标：`el-icon-link`
- 文案：`添加图片链接`
- 样式：`size="small"`、`type="primary"`、`plain`

### 2. 弹窗交互

使用 Element UI `el-dialog` 承载 URL 输入：

- **标题：** `添加图片链接`
- **内容：**
  - `el-input`，placeholder 为 `https://example.com/avatar.png`
  - 提示文案：`支持 jpg、png、gif、webp、svg 等常见图片格式。`
- **底部按钮：** `取消`、`确认添加`

**行为：**

- 点击「确认添加」时：
  - 如果输入为空或格式明显不是图片 URL，弹窗内显示错误：「请输入有效的图片链接」。
  - 校验通过后将 URL 推入 `randomAvatar` 或 `randomCover` 数组，关闭弹窗，清空输入框。
- 点击「取消」或关闭弹窗时，仅关闭弹窗，不清空输入框。
- 不再使用回车或失焦直接添加。

**复用：**

通过一个公共方法控制弹窗：

```js
showAddUrlDialog(type) {
  this.addUrlType = type // 'avatar' | 'cover'
  this.addUrlDialogVisible = true
}
```

### 3. 卡片内布局

「随机头像」与「随机封面」两个卡片采用同构布局：

```
┌─ 随机头像 ─────────────────────────────┐
│                                         │
│  [图片网格，空时显示「暂无图片…」]        │
│                                         │
│  [+ 添加图片链接]                       │
│                                         │
│  ─────────── 或 ───────────             │
│  上传本地图片（最多 5 张，每张 ≤1M）      │
│                                         │
│  [uploadPicture 拖拽上传区]             │
│                                         │
└─────────────────────────────────────────┘
```

- 按钮位于图片网格下方。
- 按钮与上传区之间用虚线分隔线分区。
- 上传区上方显示灰色小标题：`或上传本地图片（一次最多 5 张，每张不超过 1M）`。

### 4. 空状态提示

当 `randomAvatar` / `randomCover` 为空数组时，图片网格区域显示：

> 暂无图片，点击下方按钮添加。

### 5. 校验规则

基础校验（仅作为友好提示，不阻塞后端提交）：

- 非空。
- URL 以 `http://` 或 `https://` 开头。
- 后缀为常见图片格式之一（`jpg|jpeg|png|gif|webp|svg`）。

校验失败时，弹窗内 `el-form-item` 显示红色错误信息。

## 数据流

- 页面加载时调用 `getWebInfo()` 拉取 `randomAvatar`、`randomCover` 等数据。
- 点击「确认添加」后，向对应数组 `push` 新 URL。
- 点击「保存」时调用 `saveRandomResources()`，提交：

```js
{
  id: this.webInfo.id,
  randomName: JSON.stringify(this.randomName),
  randomAvatar: JSON.stringify(this.randomAvatar),
  randomCover: JSON.stringify(this.randomCover)
}
```

- 后端 API 路径和请求方式保持不变。

## 视觉规范

- 弹窗宽度：`420px`。
- 按钮：`size="small"`、`type="primary"`、`plain`。
- 上传区标题：字号 12px，颜色 `#909399`。
- 分隔线：`border-top: 1px dashed #dcdfe6; padding-top: 16px; margin-top: 16px;`。
- 空状态提示：字号 14px，颜色 `#909399`，居中显示。

## 兼容性

- 保持 Vue 2 语法，不引入 Vue 3 特性。
- 保持 Element UI 组件，不引入新的 UI 库。
- 保持现有 API 路径和请求方式不变。

## 验收标准

- [ ] 「随机头像」和「随机封面」卡片内按钮文案为「添加图片链接」，带链接图标。
- [ ] 点击按钮弹出标题为「添加图片链接」的对话框。
- [ ] 输入合法 URL 并确认后，URL 出现在对应图片网格中。
- [ ] 输入空/非法 URL 时弹窗内显示错误提示，不关闭弹窗。
- [ ] 上传区上方显示说明标题。
- [ ] 保存随机资源时，新添加的 URL 能正常提交到后端。
- [ ] 原有本地上传功能无回归。
- [ ] 随机名称模块不受本次改动影响。

## 风险与回滚

- **风险：** 弹窗引入后，原来通过回车/失焦快速添加的习惯需要适应。处理：文案和按钮已足够清晰，弹窗确认也减少了误操作。
- **回滚：** 如遇到问题，可恢复原来的内联输入框展开逻辑，保留新文案作为过渡。
