# webEdit 后台管理页面布局优化设计

## 背景

`liuliupi-ui/src/components/admin/webEdit.vue` 是后台管理中的「网站设置」页面。当前页面将 5 个功能分组（基础信息、公告、随机名称、随机头像、随机封面）沿垂直方向顺序堆叠，导致页面过长、视觉层次弱、图片上传区域占用空间过大。

## 目标

1. 让分组之间的层级关系更清晰。
2. 让图片上传区域更紧凑、合理。
3. 在保持现有 Element UI 技术栈的前提下，让整体视觉风格更现代。

## 约束

- 项目使用 Vue 2 + Element UI。
- 保留现有后端 API 调用逻辑，尽量降低改动风险。
- 保留 URL 输入框 + 上传按钮的图片编辑方式。

## 方案

采用 **Element UI 原生组件重构** 方案。

### 标签页组织

使用 `el-tabs` 将 5 个原分组收纳为 3 个标签页：

| 标签页 | 包含内容 |
|---|---|
| 基础信息 | 网站名称、网站标题、页脚、状态、背景、头像、提示 |
| 公告 | 公告 tag 列表与编辑 |
| 随机资源 | 随机名称、随机头像、随机封面 3 个子模块 |

### 基础信息标签页

- 使用 `el-form` 水平标签布局（`label-width="100px"`）。
- 背景、头像字段改为紧凑行：
  - URL 输入框（占主要宽度）
  - 右侧 40×40 缩略图
  - 右侧「上传」按钮
- 提示字段保持文本域，可编辑/禁用切换保留。
- 保存按钮统一为「保存」，放置在表单右下角。

### 公告标签页

- 保留 `el-tag` 列表 + 输入框的交互方式。
- 保存按钮统一为「保存」，放置在右下角。

### 随机资源标签页

- 页面内使用 `el-card` 或带边框容器将 3 个子模块分区：
  - **随机名称**：tag 列表 + 输入框。
  - **随机头像**：图片网格（带删除），底部保留 `uploadPicture` 上传入口。
  - **随机封面**：图片网格（带删除），底部保留 `uploadPicture` 上传入口。
- 三个子模块共用一个「保存」按钮，放置在页面右下角，一次提交 `randomName`、`randomAvatar`、`randomCover`。

### 组件提取

新增 `ImageUrlInput` 组件，封装以下结构：

```
[label]
  ├── URL 输入框
  ├── 缩略图 el-image
  └── 上传按钮 → uploadPicture
```

该组件用于背景、头像字段，保持图片输入逻辑一致，便于复用。

## 数据流

- 页面加载时调用 `getWebInfo()` 拉取全部数据。
- 切换标签页时不重新加载，所有数据保持在同一 Vue 实例的 `data` 中。
- 各标签页保存时调用现有 `updateWebInfo(param)`，仅提交当前标签页相关的字段：
  - 基础信息：提交 `webInfo` 完整对象（与现有逻辑一致）。
  - 公告：提交 `{ id, notices }`。
  - 随机资源：提交 `{ id, randomName, randomAvatar, randomCover }`。

## 视觉规范

- 标签页使用 Element UI 默认样式。
- 表单区域最大宽度限制在约 900px，避免在大屏上过度拉伸。
- 随机资源子模块使用 `el-card` 或等效边框容器，间距 20px。
- 保存按钮统一使用 `type="primary"`，右下角对齐。

## 兼容性

- 保持 Vue 2 语法，不引入 Vue 3 特性。
- 保持 Element UI 组件，不引入新的 UI 库。
- 保持现有 API 路径和请求方式不变。

## 验收标准

- [ ] 页面展示 3 个标签页：基础信息、公告、随机资源。
- [ ] 基础信息中背景、头像字段为 URL 输入 + 缩略图 + 上传按钮的紧凑行。
- [ ] 所有保存按钮文案为「保存」。
- [ ] 随机资源页内只有一个「保存」按钮。
- [ ] 各标签页保存后提示成功，数据刷新正常。
- [ ] 现有功能（上传、删除 tag、开关状态等）无回归。

## 风险与回滚

- 风险：标签页切换可能导致表单验证状态重置。处理：每个标签页独立使用 `ref` 管理表单。
- 回滚：如遇到问题，可回退到使用 `el-tabs` 包裹原内容的过渡版本。
